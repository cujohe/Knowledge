## 3.1 C 是什么？

如果你只记住一句话：**C 是一门「可移植的汇编语言」**。

它离硬件只有一层薄纱。你写的几乎每一行 C 代码，都能在脑海中翻译成对应的 CPU 操作。这既是 C 的诅咒（太底层了，写起来累），也是 C 的魔力（没有黑箱，一切透明）。

C 的抽象层级——把它放在我们之前画的「抽象阶梯」上：

```
高层：Python / JavaScript —— 不需要关心内存
        ↑
中层：Java / C# —— JVM/CLR 管理内存，但不让你碰指针
        ↑
底层：C —— 可以看到内存，可以操作内存，必须管理内存
        ↑
最底层：汇编 —— 直接操作寄存器和指令
```

## 3.2 C 的内存模型——最重要的一节

不理解内存模型就不算真正理解 C。我们把它掰开揉碎。

### 物理视角

你的程序运行时，操作系统给了它一块**虚拟地址空间**——本质上是一个超大的字节数组，从地址 0 到地址 N（32 位系统约 4GB，64 位系统巨大）。

```
内存地址    内容
─────────────────
0x0000      [   ]
0x0001      [   ]
...         ...
0x7fff      [   ]
```

每一个地址指向一个**字节**（8 个 bit）。C 让程序员可以直接操作这些地址——这就是指针。

### 逻辑视角：四个区

C 程序的内存被编译器+操作系统分成四个区域：

```
高地址
┌─────────────┐
│    栈        │  ← 局部变量、函数参数。自动管理，函数返回即释放
│   Stack     │     分配极快（一条 CPU 指令），但空间有限（通常 8MB）
├─────────────┤
│     ↓       │  栈向下增长
│     ↑       │  堆向上增长
├─────────────┤
│    堆        │  ← malloc/free 管理。程序员手动控制生命周期
│   Heap      │     空间大（几乎等于剩余 RAM），但需要手动释放
├─────────────┤
│  静态数据区   │  ← 全局变量、static 变量。程序启动时分配，结束时释放
│  Data/BSS   │
├─────────────┤
│  代码区      │  ← 程序的机器码，只读
│   Text      │
└─────────────┘
低地址
```

这四块区域对理解 C（及 C++、Rust）至关重要。来看一段代码和它在内存中的分布：

```c
#include <stdlib.h>

int global_var = 42;           // 静态数据区（已初始化）

int main() {
    int local = 10;            // 栈
    int* p = malloc(100);      // p 本身在栈上，但它指向的内存（100 字节）在堆上
    *p = 5;                    // 往堆上的那块内存写数据
    // ... 用完了
    free(p);                   // 释放堆内存
    return 0;
}                               // local 自动释放（离开栈帧）
```

### 为什么需要堆？

栈虽然快，但它有两个致命限制：

1. **空间有限**：默认只有几 MB。你要存一个 500MB 的视频文件？栈直接被撑爆。
2. **生命周期固定**：函数返回，栈上的变量就没了。如果你想「创建一个东西，等用完了再销毁」——只能放堆上。

堆的本质是：**从巨大的空闲内存池里，由程序员手动申请一块，用完再还回去。**

## 3.3 指针——C 的灵魂

指针就是**一个存地址的变量**。

```c
int x = 42;        // x 是一个整数，存在内存某个地址上
int* p = &x;       // p 是一个指针，存的是「x 的地址」
                   // & 是「取地址」操作符

printf("%d\n", x);   // 输出 42
printf("%d\n", *p);  // 也输出 42！* 是「解引用」——顺着地址找到值
```

### 指针 vs 普通变量

```
普通变量 int x = 42：
  内存某处存着一个值 42

指针变量 int* p = &x：
  内存某处存着一个地址（指向 x 的位置）
```

### 指针和数组

C 中最微妙的等价关系：

```c
int arr[5] = {1, 2, 3, 4, 5};
int* p = arr;        // 数组名退化为指向第一个元素的指针

arr[2]  ==  *(arr + 2)   // 数组索引本质就是指针运算
p[2]    ==  *(p + 2)     // 指针也可以用 [] 语法
```

`arr[2]` 被编译器翻译成「从 arr 地址开始，往后偏移 2 个 int 的大小，取出那里的值」。

这就是为什么 C 数组不做越界检查——编译器根本不知道你传的索引是否合法。它只是忠实地执行你的指令：「从那个地址往后 200 个字节，把那里的值读出来」——即使那个地址不属于你。

### 指针的指针

```c
int x = 42;
int* p = &x;      // 指向 int 的指针
int** pp = &p;    // 指向「指向 int 的指针」的指针

**pp = 100;       // 把 x 改成 100（两次解引用）
```

这在参数传递中很有用：如果你想让一个函数修改外部的指针，就得传指针的指针。

## 3.4 核心语法

### 基本类型

```c
char      c = 'A';    // 1 字节，通常用于 ASCII 字符
int       i = 42;     // 通常 4 字节，整数
float     f = 3.14f;  // 4 字节，单精度浮点
double    d = 3.14;   // 8 字节，双精度浮点
unsigned int u = 100; // 无符号整数，不能表示负数，但正数范围翻倍
```

没有 `string` 类型。字符串是 `char` 数组，以 `\0` 结尾：

```c
char name[] = "Alice";   // 实际存储：'A' 'l' 'i' 'c' 'e' '\0'
```

### 结构体

```c
struct Point {
    int x;
    int y;
};

struct Point p1 = {10, 20};
p1.x = 30;               // 通过 . 访问成员

struct Point* pp = &p1;
pp->y = 40;              // 通过指针访问用 -> （等价于 (*pp).y）
```

### 控制流

跟大多数语言差不多：

```c
if (x > 0) {
    // ...
} else if (x == 0) {
    // ...
} else {
    // ...
}

for (int i = 0; i < 10; i++) {
    // ...
}

while (condition) {
    // ...
}

switch (value) {
    case 1: /* ... */ break;
    case 2: /* ... */ break;
    default: /* ... */
}
```

### 函数

```c
// 声明（告诉编译器存在这个函数）
int add(int a, int b);

// 定义（实际的函数体）
int add(int a, int b) {
    return a + b;
}
```

C 的参数传递是**按值传递**（pass by value）：函数收到的是实参的一个副本。要修改外部变量，必须传指针：

```c
void increment(int* p) {
    (*p)++;   // 通过指针修改外部变量
}

int x = 5;
increment(&x);  // x 现在等于 6
```

## 3.5 预处理器

在编译器之前运行——真正的「代码之前先处理代码」：

```c
#include <stdio.h>        // 把 stdio.h 的内容原样插入此处
#define MAX_SIZE 100      // 宏：编译前把所有 MAX_SIZE 替换为 100
#define SQUARE(x) ((x)*(x))  // 宏函数（要注意括号！）

#ifdef DEBUG
    printf("调试模式\n");
#endif
```

宏是「文本替换」而非真正的函数。`SQUARE(1+2)` 展开为 `((1+2)*(1+2))` = 9，而不是你想要的 9……等等，这个例子恰好正确。但 `SQUARE(a++)` 就灾难了——展开成 `((a++)*(a++))`，a 被加了两次。

## 3.6 编译全景

回看第一部分讲过的四步：

```
hello.c
  ↓ gcc -E          预处理器 → hello.i（展开所有 # 指令）
  ↓ gcc -S          编译器   → hello.s（汇编代码）
  ↓ gcc -c          汇编器   → hello.o（目标文件）
  ↓ gcc             链接器   → hello（可执行文件）
```

实际命令：

```bash
# 一步到位
gcc hello.c -o hello

# 分步，观察中间产物
gcc -E hello.c -o hello.i        # 看预处理结果
gcc -S hello.c -o hello.s        # 看汇编代码
gcc -c hello.c -o hello.o        # 只编译不链接
gcc hello.o -o hello             # 链接成可执行文件
```

## 3.7 标准库

C 标准库很小，核心的几个：

| 头文件 | 功能 |
|:---|:---|
| `<stdio.h>` | 输入输出：printf、scanf、fopen、fread |
| `<stdlib.h>` | 杂项：malloc、free、atoi、exit、qsort |
| `<string.h>` | 字符串操作：strlen、strcpy、strcmp、memcpy |
| `<math.h>` | 数学函数：sin、cos、sqrt、pow |
| `<time.h>` | 时间函数：time、clock、sleep |
| `<assert.h>` | 调试断言：assert(condition) |

## 3.8 C 的工具链

| 工具 | 做什么 |
|:---|:---|
| **gcc / clang** | 编译器 |
| **make / cmake** | 构建系统（管理多个 .c 文件的编译顺序） |
| **gdb / lldb** | 调试器（设置断点、查看变量、单步执行） |

| **strace / ltrace** | 追踪系统调用和库函数调用 |

## 3.9 从入门到熟练：C 语言学习路径

### 阶段一：能读能写（2-4 周）

- 基本类型、控制流、函数
- 数组和字符串（char[]）
- 会用 printf/scanf
- 能写 200 行以内的小程序（计算器、文字游戏）

**关键练习**：实现一个简单的命令行通讯录，支持增删查改。

### 阶段二：理解内存（4-8 周）

- 彻底搞懂指针：* & -> 三个操作符
- 指针和数组的关系
- 动态内存：malloc / free / realloc
- 栈 vs 堆的分配策略
- 结构体 + 指针 = 链表、树

**关键练习**：手动实现一个动态数组（类似 C++ 的 vector），支持自动扩容。

### 阶段三：工程实践（持续）

- 多文件项目的组织方式（头文件、实现文件、Makefile）
- 用 gdb 调试
- 用 valgrind 检查内存泄漏
- 理解 const、static、extern 的作用
- 函数指针（回调模式）
- 了解未定义行为（UB）：数组越界、整数溢出、空指针解引用

**关键练习**：读一个开源 C 项目的源码（推荐：Redis 的网络模块或 SQLite 的简单部分），理解真实项目怎么组织。

### 阶段四：系统编程（进阶）

- 文件 I/O（open/read/write vs fopen/fread/fwrite）
- 进程与线程（fork、pthread）
- 网络编程（socket、bind、listen、accept）
- 信号处理

这是 C 的**主场**——操作系统的接口本身就是 C。

## 3.10 C 在什么领域仍然不可替代？

| 领域 | 为什么是 C |
|:---|:---|
| **操作系统内核** | Linux、Windows、macOS 内核都是 C（和少量汇编写） |
| **嵌入式系统** | 资源受限设备（单片机）上，C 是唯一高效的语言 |
| **数据库引擎** | Redis、SQLite、PostgreSQL 核心都是 C |
| **语言运行时** | Python（CPython）、Ruby（CRuby）、PHP 的解释器都是 C |
| **网络协议栈** | Nginx、HAProxy 等高性能网络中间件 |
| **固件/驱动** | 硬件直接打交道，必须精确控制内存和设备寄存器 |
