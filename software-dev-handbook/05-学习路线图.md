## 5.1 元技能：如何学习一门编程语言

在给你具体路线之前，先掌握「学习方法」：

### 不要背语法，要理解「为什么这样设计」

每当你遇到一个新语法特性时，问自己三个问题：

1. **它解决了什么问题？**（这个特性出现之前，程序员是怎么做的？）
2. **它引入了什么代价？**（性能？复杂度？限制？）
3. **它和其他语言中类似的东西有什么不同？**

例如：为什么 Rust 有 `Option` 而不是 `null`？

- 问题：Tony Hoare 把 null 称为「十亿美元的错误」——空指针异常是最常见的运行时崩溃
- 代价：必须用模式匹配或 `unwrap()` 处理，代码更啰嗦
- 对比：Java 的 `Optional` 是可选的（没人强制你用），Rust 的 `Option` 是强制的

### 学编程 = 学概念 + 练手感

- **概念**靠读书和看文档。你这本教材覆盖了大部分核心概念
- **手感**靠敲代码。没有捷径——必须亲手写，亲手 debug，亲手被折磨
- **比例**：30% 时间读，70% 时间写

### 一个有效的学习循环

```
读概念（20 分钟）
  ↓
写代码（40 分钟）——不要 copy-paste，手动敲
  ↓
遇到 bug → 自己 debug（15 分钟）
  ↓ 解决不了
查文档/Stack Overflow → 理解为什么这样解决
  ↓
梳理笔记（10 分钟）——用自己的话写下来
  ↓
下一个概念
```

## 5.2 三条路径：根据你的目标选择

不同的人学编程的目标不同。以下是三条不同的学习路径：

### 路径 A：全栈业务开发者（最通用）

**目标**：能独立开发完整的 Web 应用，从数据库到用户界面。

总耗时约 5-6 个月，分 5 个阶段从 Python 入门到全栈应用上线。

### 路径 B：AI / 数据科学方向

**目标**：能做数据分析、训练模型、部署 AI 应用。

总耗时约 5-6 个月，从 Python 基础到模型部署。

### 路径 C：系统 / 底层方向

**目标**：理解计算机底层原理，能写高性能、高可靠性的系统软件。

总耗时约 6-8 个月。这条路最长也最硬核。

## 5.3 通用入门路径

如果你还不知道想做什么：

```
                        Python 基础（1 个月）
                               │
           ┌───────────────────┼───────────────────┐
    对界面感兴趣          对数据感兴趣          对原理感兴趣
           │                   │                   │
    HTML/CSS/JS          NumPy/pandas           C 语言
           │                   │                   │
       → 路径 A            → 路径 B            → 路径 C
```

关键是：**不要在选择方向上花太多时间。先动手，在实践中发现自己喜欢什么。**

## 5.4 如何「从业务角度读懂代码」

### 第一步：先看业务文档，不看代码
1. README → 2. API 文档 → 3. 数据库 schema → 4. 配置文件

### 第二步：追踪一个请求的完整生命周期
从用户点击按钮，到前端发起请求，到后端处理，到数据库写入，全程跟一遍。

### 第三步：用「如果」测试理解
- 「如果订单金额是负数会怎样？」
- 「如果数据库连接断了会怎样？」

### 第四步：自己改一个功能
改 10 个功能换来的理解，胜过读 100 页书。
