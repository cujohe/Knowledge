# 第二章：变量与内存

## 2.1 变量就是贴标签

想象计算机内存是一面巨大的墙，上面有几十亿个小格子。每个格子有一个门牌号（地址），能存一个数字。

```python
age = 25
```

这一行代码在底层做的事：

```
内存墙：
地址 0x7fff... |  25  ← Python 找了块空闲内存，存 25
地址 0x7fff... |  ... 
地址 0x7fff... |  ...

你的代码里：
age  →  指向那个存了 25 的内存地址
```

**`age = 25` 不是「把 25 放进 age 这个盒子」**——更准确的理解是：

> 创建一个整数对象 `25`，放在内存里；然后让名字 `age` 指向它。

## 2.2 一张图理解 Python 的变量模型

```python
a = 10
b = a       # b 指向 a 指向的东西，不是复制一份
a = 20      # a 现在指向新的东西，b 不受影响
print(b)    # ?
```

答案是 **10**。因为 Python 的变量是**标签**，不是**盒子**：

```
执行 a = 10   →  a ──→ [10]
执行 b = a    →  a ──→ [10] ←── b
执行 a = 20   →  a ──→ [20]   b ──→ [10]  （b 没变！）
```

> 💡 **核心认知**：Python 里的 `=` 不是「放进去」，而是「重新贴标签」。这跟 C/Java 完全不同。

## 2.3 变量命名的规则

```python
name = "Alice"       ✅ 小写字母
user_name = "Bob"    ✅ 下划线分隔（蛇形命名法，Python 官方推荐）
_age = 30            ✅ 可以以下划线开头
NAME = "CONSTANT"    ✅ 全大写表示常量（约定，不是强制）
user2 = "ok"         ✅ 可以含数字（不能以数字开头）

2user = "no"         ❌ 不能以数字开头
user-name = "no"     ❌ 减号是运算符
class = "no"         ❌ class 是 Python 的关键字
```

## 2.4 重新赋值的底层逻辑

```python
x = 100        # 在内存里创建一个整数 100，标签 x 贴上
x = 200        # 在内存里创建另一个整数 200，标签 x 从 100 撕下来，贴到 200 上
x = x + 1      # 计算 x+1 得到 201，创建新整数 201，标签 x 重新贴上
```

每次赋值都**创建一个新对象**。原来的对象如果没有任何标签指向它，就会被 Python 的**垃圾回收器**自动清理——就像没人要的废纸被扫掉。

> 🧠 **垃圾回收（Garbage Collection）**：Python 会自动追踪每个对象被多少个变量引用。引用数降到 0，内存自动释放。**你几乎不需要手动管理内存。**

## 2.5 `id()` 和 `is`：看见内存

Python 给了你两个工具来「看见」内存：

```python
a = 100
b = 100

print(id(a))    # 140725838123456  ← a 指向的内存地址
print(id(b))    # 140725838123456  ← 咦，和 a 一样！
print(a is b)   # True  ← "a 和 b 指向同一个对象吗？"
```

**小整数的秘密**：Python 启动时，把 -5 到 256 这些常用数字**预先创建好**了。所以 `a = 100` 和 `b = 100` 指向的是同一个预制的 100。

```python
a = 1000
b = 1000

print(id(a))    # 140725838999888
print(id(b))    # 140725838999920  ← 不一样了！
print(a is b)   # False
```

> 💡 别依赖这个优化写代码。**用 `==` 比较值，用 `is` 比较「是不是同一个对象」。**

## 2.6 一个实战：交换两个变量

```python
# 经典方法：用临时变量
a = 5
b = 10

temp = a   # 暂存 a
a = b      # a 换成 b
b = temp   # b 换成暂存的 a
print(a, b)  # 10 5
```

```python
# Python 独有：元组解包
a = 5
b = 10
a, b = b, a      # 一行搞定！
print(a, b)      # 10 5
```

底层原理：`b, a` 先创建一个元组 `(10, 5)`，然后 `a, b = (10, 5)` 把两个值分别贴到 a 和 b 上。**没有创建临时变量。**
